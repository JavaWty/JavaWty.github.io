流概述

how2j

当不同的介质之间有数据交互的时候，JAVA就使用流来实现。
数据源可以是文件，还可以是数据库，网络甚至是其他的程序

比如读取文件的数据到程序中，站在程序的角度来看，就叫做输入流
输入流： InputStream
输出流：OutputStream

文件对象File

常用方法

  • Files.exists()：检测文件路径是否存在。
  • Files.createFile()：创建文件。
  • Files.createDirectory()：创建文件夹。
  • Files.delete()：删除一个文件或目录。
  • Files.copy()：复制文件。
  • Files.move()：移动文件。
  • Files.size()：查看文件个数。
  • Files.read()：读取文件。
  • Files.write()：写入文件。

字节流

输入 InputStream

InputStream是字节输入流，同时也是抽象类，只提供方法声明，不提供方法的具体实现。
FileInputStream 是InputStream子类，以FileInputStream 为例进行文件读取

  1. 用子节读取文件内容,读取到的是ASCII码,即读取到的二进制或者十进制数字
  2. 读取时需要创建子节数组

输出 OutputStream

OutputStream是字节输出流，同时也是抽象类，只提供方法声明，不提供方法的具体实现。
FileOutputStream 是OutputStream子类，以FileOutputStream 为例向文件写出数据

注: 如果文件h:/2.txt不存在，写出操作会自动创建该文件。

字符流

Reader字符输入流
Writer字符输出流
专门用于字符的形式读取和写入数据

可以读取汉字

输入 FileReader

FileReader 是Reader子类，以FileReader 为例进行文件读取

输出 FileWriter

FileWriter 是Writer的子类，以FileWriter 为例把字符串写入到文件

使用String类型的toCharArray()方法,可以将字符串转换为char数组

缓存流

字节流和字符流在每一次读写的时候，都会访问硬盘。 如果读写的频率比较高的时候，其性能表现不佳。

缓存流在读取的时候，会一次性读较多的数据到缓存中，以后每一次的读取，都是在缓存中访问，直到缓存中的数据读取完毕，再到硬盘中读取。

缓存流在写入数据的时候，会先把数据写入到缓存区，直到缓存区达到一定的量，才把这些数据，一起写入到硬盘中去。按照这种操作模式，就不会像字节流，字符流那样每写一个字节都访问硬盘，从而减少了IO操作

缓存流必须建立在一个存在的流的基础上

输入 BufferedReader

BufferedReader需要先创建字符流FileReader

输出 BufferedWriter

BufferedWriter缓存字符输出流， 可以一次写出一行数据

输出 PrintWriter

推荐使用PrintWriter

PrintWriter和BufferedWriter

  1. PrintWriter的print、println方法可以接受任意类型的参数，而BufferedWriter的write方法只能接受字符、字符数组和字符串；

  2. PrintWriter的println方法自动添加换行，BufferedWriter需要显示调用newLine方法；

  3. PrintWriter的方法不会抛异常，若关心异常，需要调用checkError方法看是否有异常发生；

  4. PrintWriter构造方法可指定参数，实现自动刷新缓存（autoflush）；

  5. PrintWriter的构造方法更广。

    数据流

要用DataInputStream 读取一个文件，这个文件必须是由DataOutputStream 写出的，否则会出现EOFException，因为DataOutputStream 在写出的时候会做一些特殊标记，只有DataInputStream 才能成功的读取。

总结

  1. strem结尾的是字节流,读取到的是二进制文件
  2. read结尾的是字符流,可以读取中文

按功能来分：输入流（input）、输出流（output）。

按类型来分：字节流和字符流。

字节流和字符流的区别是：字节流按 8 位传输以字节为单位输入输出数据，字符流按 16 位传输以字符为单位输入输出数据。
