总结

  1. springboot里面存在了大量的XXXAutoConfiguration文件,这些文件自动装配了一个XXXProperties文件
  2. 因为XXXProperties文件有默认值,所以也导致了XXXAutoConfiguration是有默认值的
  3. 我们需要修改这些默认值,所以就按照``XXXProperties的前缀名那样,更改属性,用来和XXXProperties`绑定

缓存

缓存的使用

  1. 引入spring-boot-starter-cache模块
  2. @EnableCaching开启缓存
  3. 使用缓存注解
  4. 切换为其他缓存

缓存注解

@EnableCaching

在启动类上添加@EnableCaching,表示启用缓存

@Cacheable

主要针对方法配置，能够根据方法的请求参数对其进行缓存

@CachePut

既调用方法,又更新缓存

  1. 先调用方法
  2. 然后将目标方法的结果缓存起来

  • 查询的时候,传参如果是id,那么在添加了@CachePut注解的方法上,也要声明一下key为id,

  • 声明key有两种方式:

    • #employee.id
    • #result.id

@CacheEvict

  • key: 根据指定的id清空缓存
  • allEntries = true 清空所有缓存,默认false
  • beforeInvocation = true ,默认是false,true表示执行在方法之前,如果方法发生了异常,也会清除掉缓存

@Caching

此注解,抽取缓存的公共配置
