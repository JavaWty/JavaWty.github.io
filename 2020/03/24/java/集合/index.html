几种集合之间的关系

  1. Queue队列: 先进先出队列

Deque: 双向链表

Stack栈: 先进后出

注：Collection和Map之间没有关系，Collection是放一个一个对象的，Map 是放键值对的

注：Deque 继承 Queue,间接的继承了 Collection

  1. LinkedList和ArrayList关系

LinkedList， 插入，删除数据快

ArrayList是顺序结构，所以定位很快

  1. ArrayList和HashSet

ArrayList: 有顺序 HashSet: 无顺序

List中的数据可以重复 Set中的数据不能够重复

HashSet： 无序

LinkedHashSet： 按照插入顺序

TreeSet： 从小到大排序

  1. HashMap和Hashtable都实现了Map接口，都是键值对保存数据的方式

区别1：

HashMap可以存放 null

Hashtable不能存放null

区别2：

HashMap不是线程安全的类

Hashtable是线程安全的类

常用方法

List方法

add 1.增加对象 2.指定位置增加对象
contains 判断是否存在,判断的是对象
get 获取指定下角标位置的对象
indexOf 获取对象所处的位置
remove 1.删除对象 2.删除指定位置对象
set 替换指定位置的元素
size 获取大小
toArray 转换为数组
addAll 把另一个容器所有对象都加进来
clear 清空

ArrayList

因为ArrayList实现了List接口,所有List接口有的方法,ArrayList也有

LinkedList

LinkedList 实现了List接口,还实现了双向链表结构Deque接口,还实现了队列Queue接口

  1. 首先,需要明确,序列分先进先出FIFO,先进后出FILO
    FIFO在Java中又叫Queue 队列
    FILO在Java中又叫Stack 栈

  2. Deque是双向链表结构,可以很方便地在头尾添加删除数据

    addFirst ()在最前面插入

    getFirst() 查看最后一位

    addLast () 在最后插入

    removeFirst() 移除最前面的

    removeLast() 移除最后面

  3. Queue是先进先出队列 FIFO，常用方法：
    offer 在最后添加元素
    poll 取出第一个元素
    peek 查看第一个元素
