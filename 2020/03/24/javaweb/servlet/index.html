Servlet

简介

客户端 —> 服务器 请求 HttpServletRequest

服务端 —> 客户端 响应 HttpServletResponse

生命周期

访问Servlet方法的时候

  1. 创建一个Servlet实例对象
  2. 调用nit方法 初始化
  3. 调用service方法 处理请求
  4. 以后的请求,只调用service方法,getServletConfig—getServletInfo手动调用
  5. 服务器停止或者项目卸载,调用destory方法

HttpServlet

实现原理图示

原理说明

  1. 发送 http://localhost:8080/XXX请求

  2. Servlet是父接口,优先实现,调用service (ServletRequest req, ServletResponse res) 实现,但是Servlet是接口,没有实现方法,被GenericServlet类实现

  3. GenericServlet类继承Servlet接口,调用本身的 abstract void service(ServletRequest req, ServletResponse res) 抽象方法,本身并没有实现具体方法

  4. HttpServlet实现抽象类GenericServlet,并实现GenericServlet的service抽象方法,写出了具体方法service(ServletRequest req, ServletResponse res)

service方法,强转ServletRequest 为 HttpServletRequest ,强转ServletResponse 为 HttpServletResponse,

最后调用了自身的 service(HttpServletRequest req, HttpServletResponse resp)

  1. HttpServlet中的service(HttpServletRequest req, HttpServletResponse resp)方法首先获取请求方式,判断是那种请求,就会调用哪种方法,例如:是Get请求,就返回doGet(HttpServletRequest req, HttpServletResponse resp).

HttpServlet中的service(HttpServletRequest req, HttpServletResponse resp)方法

  1. HttpServlet中的 doGet方法有自己的实现方式,我们编写的Servlet继承HttpServlet,对doGet方法进行重写

自定义Servlet

  1. 继承HttpServlet

  2. 重写doGet方法

Cookie

狂神说

尚硅谷

http是无状态的协议,无法识别每个用户的请求,在浏览器的开发者模式中,只能看到浏览器的相关信息,看不到用户相关信息

  1. cookie是服务器发送给客户端的,数据保存在客户端

  2. 服务端通过调用resp.addCookie(cookie);方法发送一个cookie,

    客户端通过调用req.getCookies();获取服务端发送的cookie

  3. 服务器第一次发送Cookie后,客户端会接收到Cookie,以后每次请求服务器,都会在Request Header中携带Cookie,

  4. cookie的生命周期

    • 默认生命时间:浏览器不关闭,cookie一直存在
    • 正数:表示该Cookie将在经过许多秒后过期, 过期后,浏览器会自动删除cookie
    • 0: 删除cookie
    • 负数:cookie的默认值,表示未存储cookie

Session

尚硅谷

  1. session是一种服务器端保存数据的技术,session就是当前会话的对象,可以共享当前会话的数据

  2. 服务器会为每个会话(服务器重新开启,浏览器切换都称为一次会话)创建独立的session,会有一个唯一id

  3. 服务器如何识别要使用哪一个session?

    在Request Headers中,会生成一个Cookie对象,它包含一个JSESSIONID=B8C3056F273D9813D36D3C5491A3558B,这个JessionId的值就是不同浏览器的每次新会话的session为一标识

  4. 会话控制流程

    在用户1和用户2登录的时候，我们的服务器在他们登录成功后，在session中为他们每个用户分配了一个sessionid并且存下了一个对应的信息。当用户第二次访问该服务器的时候，会将sessionid在request请求中携带者发送过去。这时我们的服务器就可以根据sessionid确定用户存储的数据，然后进行使用。

  5. 生命周期,单位秒

    • 负数 :永不过期
    • 正数 :多少秒后过期
    • 强制失效:session.invalidate();

Filter

说明

观看地址

web中的过滤器：当访问服务器的资源时，过滤器可以捋请求拦截下来，完成一些特殊的功能;

filter配置的是拦截路径,不是过滤路径;

过滤器在一次请求中,执行两遍,请求发送时是一次,请求返回时是一次;

一般用于完成通用的操作。如：登录验证、统一编码处理、敏感字符过滤.

实现自定义Filter

xml配置

注解

生命周期

  1. 服务器启动后,创建了Filter,执行了init(FilterConfig config)方法,只会执行一次一般用于加载资源;
  2. 每一次请求数据,都会执行doFilter(ServletRequest req, ServletResponse resp, FilterChain chain)方法,执行多次;
  3. 服务器只有在正常关闭的情况下,会执行destroy()方法,只执行一次,一般用于释放资源.

执行流程

  1. 发送了一个页面的请求数据,开始执行过滤器拦截,过滤器允许请求通过;

  2. 访问到了页面数据;

  3. 携带页面数据,返回到过滤器;

过滤器拦截配置

拦截路径

  1. 具体资源路径：/index.jsp 只有访问index.jsp资源时，过滤器才会被执行
  2. 拦截目录：/user/* 访问/user下的所有资源时，过滤器都会被执行
  3. 后缀名拦截：*.jsp 访问所有后缀名为jsp资源时，过滤器都会被执行
  4. 拦截所有资源：/* 访问所有资源时，过滤器都会被执行
