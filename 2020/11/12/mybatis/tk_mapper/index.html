启动类注解

在启动类上添加mapper的包扫描,需要添加的是tk.mybatis.spring.annotation.MapperScan,

而不是org.mybatis.spring.annotation.MapperScan,

不然报错tk.mybatis.mapper.MapperException:无法获取实体类com.wty.tk_mapper.bean.Employee对应的表名!

方法

XXXXPrimaryKey

此方法传入一个id时,查询的结果为NULL,

sql语句是SELECT emp_id,emp_name,emp_salary,emp_age FROM tabple_emp WHERE emp_id = ? AND emp_name = ? AND emp_salary = ? AND emp_age = ?

原因分两种情况

  1. 没有使用@Id 注解明确指定主键字段,sql语句把所有字段都当成了主键,所以每个都执行
  2. 使用@Id 主键明确标记和数据库表中主键字段对应的实体类字段。

existsWithPrimaryKey

如果需要判断一个用户名是否存在,可以在name属性上添加@Id注解,把name当成主键来测试

但是有违主键策略,慎用

insert 获取自增主键

  1. 在主键上添加注解@GeneratedValue(strategy= GenerationType.IDENTITY)并配置相应的策略
  2. 在添加数据时,通过get方法即可获取,eg:e.getEmpId()

insertSelective

非主键的字段为null时,不插入sql语句中

updateByPrimaryKeySelective

传入的参数为null时,不修改原有数据

QBC条件查询

概念

Query By Criteria
Criteria 是 Criterion 的复数形式。意思是：规则、标准、准则。在 SQL 语句中相当于查询条件。
QBC 查询是将查询条件通过 Java 对象进行模块化封装。

其他设置

注解

注解都在javax.persistence包下

@Table

当数据库表与实体类名称不一致时,用此注解绑定两个的数据

@Column

在数据表名称和实体类名称不一致时,用于绑定两个数据

@Id

使用@Id 主键明确标记和数据库表中主键字段对应的实体类字段。直接在实体类属性上添加注解

@GeneratedValue

让通用 Mapper 在执行 insert 操作之后将数据库自动生成的主键值回写到实体类对象中。

应用场景：购物车结账

  • 增加商品销量…
  • 减少商品库存…
  • 生成订单数据→封装到 Order 对象中→保存 Order 对象→数据库自动生成主键值→回写到实体类对象 Order 中
  • 生成一系列订单详情数据→List→在每一个 OrderItem 中设置Order 对象的主键值作为外键→批量保存 List

@Transient 主键

用于标记不与数据库表字段对应的实体类字段

二级缓存

实现步骤

  1. 开启缓存 mybatis.configuration.cache-enabled=true
  2. 在mapper上添加注解 @CacheNamespace
  3. 实体类实现Serializable接口
  4. 测试,对同一个查询语句进行两次查询,出现0.5即表示缓存成功,eg: Cache Hit Ratio [com.wty.mapper2.mapper.EmployeeMapper]: 0.5

复杂类型

情况概述

简单类型和复杂类型

基本数据类型 byte short int long double float char boolean
引用数据类型 类、接口、数组、枚举……
简单类型 只有一个值的类型
复杂类型 有多个值的类型

这就是一个简单类型加复杂类型

当通过id查询一个用户时,查询结果是 User(userId=1, userName=小米, address=null, season=null)

说明通用 Mapper 默认情况下会忽略复杂类型，对复杂类型不进行“从类到表”的映射。

Address实体类

概述

使用类型转换机制,进行数据的绑定

转换分为两种

  1. Address实体类->TypeHandler->字符串
  2. 字符串->TypeHandler->Address实体类

TypeHandler是一个顶级接口,它有一个实现类BaseTypeHandler,自定义转换器实现BaseTypeHandler即可

BaseTypeHandler有三个方法

解决办法

  1. 实现BaseTypeHandler抽象类,并实现四个方法

  1. 注册自定义类型转换器

    • 字段级别,在实体类上添加@ColumnType 注解

    • 全局级别

      • 在属性上添加@Column注解,使其像正常的值一样
      • 在mybatis的配置文件中配置typeHandler
